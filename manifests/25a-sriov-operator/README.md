# SR-IOV Network Operator (RoCE Only)

## Overview

This manifest deploys the **official Red Hat SR-IOV Network Operator** from OpenShift OperatorHub.

**IMPORTANT**: This operator is configured to handle **RoCE (RDMA over Converged Ethernet) devices ONLY**. InfiniBand devices are managed separately by the NVIDIA Network Operator (see `28-nvidia-network-operator`).

## Architecture

### Hybrid RDMA Approach

We use a hybrid approach to avoid the InfiniBand netlink PAGE_SIZE limitation:

1. **SR-IOV for RoCE** (this directory)
   - Uses dedicated VFs per pod
   - Full network isolation
   - Ethernet-based RDMA doesn't hit InfiniBand netlink issues

2. **NVIDIA Network Operator for InfiniBand** (`28-nvidia-network-operator`)
   - Uses RDMA shared devices (no VFs)
   - Separate resource per NIC for control
   - Avoids netlink PAGE_SIZE limit entirely

### Why This Split?

InfiniBand NICs with many VFs (16+) cause "message too long" errors when SR-IOV operator queries VF state via netlink. The kernel's ~4KB netlink response limit is exceeded.

- **RoCE devices**: Work fine with SR-IOV (no InfiniBand-specific large responses)
- **InfiniBand devices**: Use RDMA shared devices to avoid netlink queries entirely

## Components

### Subscription
- **Operator**: `sriov-network-operator`
- **Source**: `redhat-operators` (official OperatorHub)
- **Channel**: `stable`
- **Approval**: Automatic

Version is auto-selected based on OpenShift version.

### Discovery and Configuration

SR-IOV policies are auto-generated by the NIC discovery process (`26a-sriov-discovery`):
- Discovers all Mellanox/NVIDIA NICs on worker nodes
- Separates RoCE and InfiniBand devices by link layer type
- **Only creates SR-IOV policies for RoCE devices**
- Exports InfiniBand devices to configmap for NVIDIA Network Operator

## Deployment Order

1. **Wave 25**: SR-IOV Network Operator installation
2. **Wave 26**: NIC discovery and SR-IOV policy generation (RoCE only)
3. **Wave 28**: NVIDIA Network Operator configuration (IB devices)

## Removed: Custom Image Patches

Previous versions used custom-built images to work around the InfiniBand netlink issue:
- `sriov-network-operator-config-daemon:netlink-fix-v15`
- `sriov-cni:netlink-fix-v2`
- `ib-sriov-cni:netlink-fix`

These are **no longer needed** with the hybrid approach. We now use official images for RoCE and avoid SR-IOV entirely for InfiniBand.
